在Windows本地或虚拟机中安装和部署triton推理服务

本文最后更新于：16 天前

1. Docker安装

可以在Windows本地环境安装Docker Desktop也可以在虚拟机环境Ubuntu中安装，实测在Windows本地安装实际上也是会需要安装在WSL2中的Linux子系统，同时，实测在本地安装启动docker容器要比在vmware虚拟机中所占用的内存要大很多，因此，看个人情况选择安装方式。
此外，由于triton的官方镜像较大，因此，如果是在虚拟机中安装使用，建议虚拟机创建时分配大一些的磁盘空间，个人建议60G+。

1.1 Windows安装Docker Desktop

    1. 关于如何安装Windows子系统WSL2可以查看这里在Windows10中安装WSL1&2，并配置C/C++和VSCode环境 - ShuHui’s Blog (shuhuicoder.github.io)
    1. 下载Docker for windows下一步安装即可

2. triton安装部署

    1. triton官方镜像拉取Triton Inference Server | NVIDIA NGC，根据网速情况，可能需要一些时间，镜像大小大概20-30G？注意磁盘空间的占用，如果大小不够，可以更改docker镜像的默认存储位置，对于vm虚拟机则需要注意分配大一些的磁盘空间。Windows中可以在Docker Desktop中的设置进行更改。
    1. 通过镜像创建一个新容器并运行，通过一下命令将会创建一个新的容器，并进入到改容器。
      note：需要注意的是 –rm 命令选项将会在退出容器时将容器删除，如果不希望如此，则不需要带该选项！！！
    1. 拉取triton inference server的源码GitHub - triton-inference-server/server: The Triton Inference Server provides an optimized cloud and edge inferencing solution.， 当然需要在某个路径下，例如在/home目录下，
  • Note: 需要注意的是，这里开始所执行的命令都是在docker中的容器中，而不是宿主机Ubuntu。
    1. 运行server-main源码中的example示例 拉取完模型repos之后，当前目录下就会出现model_repository目录，里面可以看到相应的一些示例模型
    1. 启动triton服务 在经过一段模型加载之后，当出现如下页面，则证明模型加载成功，triton服务正确启动
    1. 外部验证服务是否正确启动
      在终端新开一个窗口，可以通过docker start -i containerID来进入容器，查看当前的容器是docker ps -a命令，或者直接在虚拟机ubuntu的终端执行，在虚拟机外部本机，则将localhost替换为虚拟机内ubuntu的ip即可。 出现如下界面则表示服务启动成功，可以正常访问
